# 設計ドキュメント — 要件定義サポートAI ver.2

このディレクトリは本アプリの**設計仕様書（Specification）**です。
[Document-Driven Development（DDD）](../../../amplifier/.claude/commands/ddd/0-help.md)の思想に基づき、
**「ドキュメントが仕様であり、コードはドキュメントが記述した内容を実装する」**という原則で記述しています。

実装を変更するときは、まずこのドキュメントを更新し、それに合わせてコードを再生成・修正してください。
ドキュメントとコードを乖離（ドリフト）させないことが目的です。

## アプリ概要

AI（Google Gemini）との対話を通じて、エンジニア初心者のシステム企画・要件定義・技術選定を支援する Web アプリ。
Google ログイン、要件書のクラウド保存、ユーザー単位の API 利用制限を備えた本番デプロイ版です。

- **フェーズ1**: 9つの観点（課題→ターゲット→ゴール→機能→非機能→完了条件→制約→スコープ外→リスク）を1問ずつ対話で整理
- **フェーズ2**: 整理した要件をもとに技術スタックを2〜3案提示
- 整理結果はリアルタイムで Markdown 化され、ダウンロード／クラウド保存できる
